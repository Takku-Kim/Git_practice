This is for my git practice.
The first txt document

Stage1
commend
1. touch text1.txt // 파일 생성
2. git status // 상태보기
3. ls // 리스트 보기
4. git add text1.txt / git add . // git 추가
5. clear
6. git log
6-1. git log -p, -1, --oneline

7. git commit /git commit -m "" // commit 생성 또는 메세지 생성
7-1. shift+i
7-2. esc
7-3. :wq

8. git init
9. git config --global user.name ""
10. git config --global user.email ""

11. git diff / 최근 commit과 현재 파일의 차이를 보여줌_잘 사용하지 않음
11-2. git difftool / vim 에디터가 나와서 보여줌
11-2-1. :q or :qa 쓰면 나와짐


Stage2
1. touch .gitignore
2. nano .gitignore -> secret.txt -> ctrl + x -> y // 파일안에 내용 작성
3. git checkout commit_hash // 특정 시점으로 파일을 돌림
4, git reflog // head 이력 확인

Stage3
1. git revert // 특정 사항의 커밋을 되돌리며, 이러한 이력도 남김
1-2. git revert hash - 그 해쉬에서부터 revert 하겠다라는 것
2. git reset file_name // stage를 내림
3. git reset hash --hard / --mixed / --soft  - 돌리고자 하는 hash로

Stage4
"Branch"
1. git branch / branch 목록
2. git branch name / name 으로 현재 branch 복사
3. git checkout name / name 으로 branch 이동
4. git merge name / 병합하고 싶은 branch name을 현재 branch로 병합
5. git branch -d name / name의 branch를 삭제

Stage5
"Github"
1. git remote add origin address / 현재 git과 online 저장소와 연결
2. git remote -v / 현재 연결 정보
3. git push / 원격 저장소에 업로드, branch와 저장소 이름을 정확하게 명시해야 함
3-1. git push -u origin master / upstream의 origin 저장소의 master branch를 명시

4. cd .. / 상위 폴더로 이동
5. git clone address / address 내용 복사

6. git pull / 원격 저장소에서 다운로드
